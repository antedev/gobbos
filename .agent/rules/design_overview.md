This is the core game philosophy. It serves as keeping the game focused on the core priciples. 

# Overview description

It is a TTRPG where the player plays Goblins. It should be a fast, fun and chaotic system. Death of a character is far from the end, rather it is almost neccessary to increase in power, like a roguelite game. They grant bonuses to the next character the player creates. 
The players also not only control their character, but are also leading mobs of other goblins into battle. 
The players can go on raids to find resources to upgrade the lair and socity they are in, which gives them access to even more powerful abilities. 
The core game mechanic is that the player rolles a dice pool of d6s, looking for a set number of successes. 
The GM do not roll dice!


# Design tenents 
The Design tenents are orderd, and it is ok to break one of the tenents if it better serves one higher up at the list even better. 

1) It should be fun at the table. 
2) There should not be any math to interpret the dice result once the dice are rolled. The player should only have to look for certain numbers or, in some cases, combination of dices. 
3) Embrace chaos and randomness, but players should with a cost, sometimes, be able to harness it. 
4) It should feel like playing goblins
5) Character death is a part of "leveling" up, but need to be balanced so that it still is relevant to keep the character alive 

# Exceptions and updates
If the rules and tenents in someway stands in the way of creating the best game possible; Make a case for that the rules should be changed. But make explicit, and wait for confirmation before doing suggestions and changes.
